# Tiger Notes

- Tiger repo: https://github.com/Thomas-Ulrich/tiger/tree/thomas/check_failure Mind the branch used

- Procedure to install Tiger: https://www.m2clab.com/tiger/installation

- current mesh was generated with gmsh and tiger_files/ExtendedBottom_1703.geo
    - lower_dimensional_block needs to be added for PhysicalNames of lower dimension.
    - before that, a SeisSol mesh is obtained by running this script: scripts/convert_msh_to_SeisSol.py 

- tiger_files/ExtendedBottom_large.geo is a much larger geometry, but the model does not converge.

- tiger inputs files, to be run with e.g. `mpirun -n 20 ../myLibs/Tiger/tiger-opt -i tiger_files/THM-criterion-ex.i`

    tiger_files/THM-criterion-ex_gravity_only.i : initializing gravity (this did not converge on the extended mesh)
    tiger_files/THM-criterion-ex.i : injection phase
    tiger_files/THM-criterion-ex_updated_add_stress_solution_user.i : adding seissol stresses (first convert seissol file to exodus with paraview)

- tiger_seissol_script.sh to be run e.g. as:

   `./tiger_seissol_script.sh /import/heisenbug-dump/ulrich/myLibs/Tiger/tiger-opt  tiger_files/THM-criterion-ex-flood.i 10 SeisSol_Release_srome_4_elastic SeisSolSetup/parameters.par`

- to run on other computer than heisenbug:

   `export LD_LIBRARY_PATH=/import/heisenbug-dump/ulrich/myLibs/moose/framework:/import/heisenbug-dump/ulrich/myLibs/moose/framework/contrib/hit:/import/heisenbug-dump/ulrich/myLibs/moose/framework/contrib/pcre:/import/heisenbug-dump/ulrich/myLibs/moose/modules/fluid_properties/lib:/import/heisenbug-dump/ulrich/myLibs/moose/modules/module_loader/lib:/import/heisenbug-dump/ulrich/myLibs/moose/modules/phase_field/lib:/import/heisenbug-dump/ulrich/myLibs/moose/modules/solid_mechanics/lib:/import/heisenbug-dump/ulrich/myLibs/Tiger/lib:/import/heisenbug-dump/ulrich/myLibs/Tiger/test/lib:$LD_LIBRARY_PATH`

(generated by running on heisebug:
   `sh scripts/add_heisenbug_libs.sh /import/heisenbug-dump/ulrich/myLibs/Tiger/tiger-opt)`
